Tools
13 commands — panels, filters, DWG floors, export, voice, utilities

The Tools panel is the largest and most diverse. It contains panel toggles, filter/classification utilities, DWG-to-Revit workflows, a JSON data exporter for the BIM Israel web app, voice control, and element creation tools. Several commands are pure event publishers that switch the dockable panel's content view.

Event-Based Panel Switching
Commands like Filter Tree, Calc Area, and Floating Toolbar don't run transactions. They publish Prism events (SwitchDockablePanelEvent, ToggleFloatingToolbarEvent) that the dockable panel ViewModel handles by swapping its active content view.
Filter Tree Ribbon: Tools → Filter Tree
Solid

What It Does

Opens the dockable panel and switches to FilterTreeView. Builds a hierarchical tree of element categories, families, types, and parameter values from the active view or current selection. Used for quick element filtering and inspection.

Workflow

  1. Click Filter Tree.
  2. Updates IRevitContextService with active document.
  3. Publishes SwitchDockablePanelEvent("FilterTree") → dockable panel swaps content.
  4. Publishes ResetFilterTreeEvent → FilterTreeViewModel rebuilds tree.
  5. Dockable pane shown if hidden.

ResetFilterTreeEvent uses ThreadOption.PublisherThread (synchronous). Tree load must complete before control returns to Revit.
Calc Area & Volume Ribbon: Tools → Calc Area & Volume
Solid

What It Does

Switches the dockable panel to ElementCalculationView and calculates area, volume, and count for selected elements across 10+ categories. Includes material breakdown (concrete, steel, etc.) and special stair handling.

Workflow

  1. Click Calc Area & Volume.
  2. Publishes SwitchDockablePanelEvent("ElementCalculation").
  3. Publishes CalculateElementsEvent → ElementCalculationViewModel recalculates.
  4. Results displayed in dockable panel: counts + area (m²) + volume (m³) per category.

Supported Categories

Walls, Floors, Beams, Columns, Foundations, Stairs (with tread/landing breakdown), Railings, Doors, Windows. Material breakdown for concrete/steel elements.

Export JSON Ribbon: Tools → Export JSON
Solid

What It Does

Exports all model elements to JSON for the BIM Israel web application. Extracts geometry, parameters, BOQ data, and Uniformat codes for 40+ categories. Validates the export against Revit schedules and generates a discrepancy report.

Export JSON — Pipeline
graph LR A["Ensure validation\nschedules exist"] --> B["User picks\nsave location"] B --> C["Extract elements\n(40+ categories)"] C --> D["Save elements.json\n+ validation.json"] D --> E["Compare export\nvs Revit schedules"] E --> F{"Discrepancies?"} F -->|None| G["✓ All verified"] F -->|Found| H["⚠ Show\ndiscrepancy report"] style A fill:#1e3a5f,color:#fff style C fill:#7c3aed,color:#fff style G fill:#166534,color:#fff style H fill:#9a3412,color:#fff

Workflow

  1. Click Export JSON.
  2. Creates BIM_Validation_* schedules if missing (for post-export comparison).
  3. Save dialog → default name: {title}_{timestamp}.json.
  4. Extracts all elements from active view, filtered by view phase.
  5. Generates validation summary (counts + totals by category/family).
  6. Saves elements JSON + validation JSON.
  7. Compares export against Revit schedule totals (tolerance: 0.1 m³).
  8. Shows results: verified count, discrepancies (top 5 by family), category breakdown.

Export Content

Each element in the JSON includes: Id, Category, FamilyName, TypeName, Level, Volume (m³), Area (m²), Length (m), BoqCode, Zone, WorkStage, SeifChoze, SubcontractorName, and more. Uses System.Text.Json with UnsafeRelaxedJsonEscaping for Hebrew character support.

Key Files

Commands/Handlers/WebAppDataExtractorHandler.cs Services/DataExtractorService.cs Models/BimElementDto.cs
No Transaction
Read-only export. Schedule creation is the only modification (if needed).
Validation
Compares export counts/volumes against Revit validation schedules. Stair runs/landings are excluded from comparison (calculated in parent).
Hardcoded Category Mappings

24 category-to-schedule-name mappings are hardcoded in the handler. If validation schedule names differ from expected, comparison silently skips those categories.

Detect Overlapping Walls Ribbon: Tools → Detect Overlaps
Solid

What It Does

Finds same-type walls occupying nearly the same physical space (stacked duplicates). Uses a 2-stage detection: 2D pre-filter (parallel check, perpendicular distance, overlap length) then 3D solid intersection via BooleanOperationsUtils. Shows results with 3 action options.

Workflow

  1. Click Detect Overlaps.
  2. Uses selected walls if any, otherwise all walls in active view.
  3. OverlappingWallsDetector.FindOverlappingWalls() runs 2-stage detection.
  4. Results dialog shows groups: overlap %, wall type, wall count, wall to keep (most hosted elements).
  5. User chooses action:
    • Highlight: Select + temporarily isolate overlapping walls.
    • Unjoin: Unjoin all geometry joins within groups.
    • Delete Duplicates: Keep wall with most hosted elements, delete others.

Perpendicular distance< 50% wall widthWalls must be close

Workflow

  2. User picks a point on a DWG import (only ImportInstance elements selectable).

Floors From DWG All Ribbon: Tools → Floors From DWG All
Solid

What It Does

Batch floor creation: user picks a DWG import, the system extracts ALL visible floor-sized face boundaries, deduplicates near-duplicates, chains loose curves into closed loops, and creates floors for all valid boundaries.

Floors From DWG All — Pipeline
graph TD A["Pick DWG import"] --> B["Select floor type\n+ level"] B --> C["Extract all geometry\n(faces + loose curves)"] C --> D["Build CurveLoops\nfrom face boundaries"] C --> E["Chain loose curves\ninto closed loops"] D --> F["Filter: area ≥ 5 m²"] E --> F F --> G["Deduplicate:\ncentroid + area match"] G --> H["Confirm with user:\nvalid / skipped / errors"] H --> I["Create all floors\nin single transaction"] style A fill:#1e3a5f,color:#fff style C fill:#7c3aed,color:#fff style G fill:#6d28d9,color:#fff style I fill:#166534,color:#fff

Workflow

  1. Click Floors From DWG All.
  2. User picks a DWG ImportInstance.
  3. Floor type/level selector dialog.
  4. Extracts all geometry: horizontal faces (normal Z > 0.5) + loose curves.
  5. Builds CurveLoops from face boundaries; chains loose curves into closed loops.
  6. Filters out loops with area < 5 m².
  7. Deduplicates: centroid within 3mm AND area within 1 sqft → duplicate.
  8. Confirmation dialog with counts (valid, skipped-small, skipped-duplicate, errors).
  9. Creates all floors in single transaction. Commits if ≥1 created; rolls back if all fail.

Piles From DWG Ribbon: Tools → Piles From DWG (split button)
Sketchy

What It Does

User picks a DWG import. The system finds all arc elements on the "fnd" (foundation) layer, then places a structural foundation family instance at each arc center with diameter set to the arc diameter.

Workflow

  1. Click Piles From DWG.
  2. User picks a DWG import instance.
  3. DwgWrapper parses arcs on the "fnd" layer.
  4. Finds OST_StructuralFoundation family type named "Auto".
  5. For each arc: places NewFamilyInstance() at arc center, sets diameter = arc diameter.
  6. Reports total placed count.

Key Files

Commands/Handlers/AddClunasFromDwgHandler.cs
Transaction
Single transaction: "Place Piles from DWG". All-or-nothing.
Hardcoded Family Name & Layer

Requires family type named "Auto" in Structural Foundations and DWG arcs on a layer named "fnd". Both are hardcoded constants. Projects using different naming conventions will get errors.

No Diameter Parameter Validation

Assumes a "Diameter" parameter exists and is writable on the foundation family. If the parameter doesn't exist, the pile is placed but sized incorrectly with no warning.

Cross-Cutting Observations
  • Three command patterns emerge: (1) event publishers that switch dockable panel content (Filter Tree, Calc Area), (2) dialog-driven workflows with transactions (Copy From Link, Filter to Legend, Detect Overlaps), and (3) interactive pick-and-create tools (Floor DWG, Create Similar, Piles).
  • DWG floor tools are the most robust — multi-stage geometry extraction with configurable tolerances, deduplication, curve chaining, and detailed error reporting. The batch variant handles edge cases (small areas, duplicates, geometry errors) gracefully.
  • The floating toolbar is the only broken command in the entire add-in. All other commands function at least at a "sketchy" level.
  • Export JSON is the most complex command — 10-step pipeline with 40+ category extraction, validation schedule creation, element-by-element geometry reading, and post-export comparison against Revit schedules. The validation step catches data loss that would otherwise go unnoticed.
  • Piles From DWG is the most fragile — hardcoded family name ("Auto"), layer name ("fnd"), and parameter name ("Diameter") make it project-specific with no configurability.
  • Copy From Link has elegant error recovery — batch-all attempt with automatic fallback to chunked copy (20/batch) with a 5-failure circuit breaker. This pattern could be applied to other batch operations.
