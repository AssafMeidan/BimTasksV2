Structure
4 commands — beams, geometry joins, floor splitting

The Structure panel handles structural element manipulation: relocating beams, joining geometry between walls and floors/beams, and splitting compound floors. The two "Join" commands share a near-identical pattern (intersection filter → pairwise join with retry) and are the most robust handlers in the panel.

Beams To Ground Ribbon: Structure → Beams To Ground
Sketchy

What It Does

Recreates selected beams at Level 0 while preserving their top elevation. Useful when beams are modeled at arbitrary levels but need to reference Level 0 with a Z-offset for structural analysis.

Workflow

  1. Select one or more structural framing elements (beams).
  2. Click Beams To Ground.
  3. Handler finds Level 0 (elevation == 0) in the document.
  4. For each beam:
    • Reads the beam's top elevation (STRUCTURAL_ELEVATION_AT_TOP).
    • Creates a new beam at Level 0 with the same curve and family type.
    • Sets Z-justification to Top, reference level to Level 0.
    • Sets Z-offset to the original top elevation (preserves visual position).
  5. Deletes all original beams in batch.
  6. Shows count summary.

Key Files

Commands/Handlers/SetBeamsToGroundZeroHandler.cs
Originals Deleted Before Verifying Replacements

The handler deletes original beams after creating replacements, but does not verify that each new beam was actually created successfully. If doc.Create.NewFamilyInstance() returns null (rare but possible), the original beam is deleted with no replacement. Should track which originals have confirmed replacements before batch-deleting.

No Try-Catch in Main Loop

Relies on null checks but has no exception handling around individual beam processing. One malformed beam geometry could abort the entire operation mid-way, leaving some beams duplicated and others not yet processed.

Join Walls & Floors Ribbon: Structure → Join Walls & Floors
Solid

What It Does

Joins geometry between all concrete walls and the selected floors, roofs, ceilings, or structural foundations. Uses ElementIntersectsElementFilter to find spatially overlapping walls, then joins each pair. Handles join-order sensitivity by retrying in reverse order on failure.

Workflow

  1. Select one or more floors, roofs, ceilings, or foundations.
  2. Click Join Walls & Floors.
  3. For each selected host element:
    • Finds all walls that intersect it geometrically.
    • Skips: already-joined pairs, thin walls (<30mm), curtain walls, stacked walls.
    • Attempts JoinGeometry(host, wall).
    • If ArgumentException, retries with switched order: JoinGeometry(wall, host).
  4. Shows detailed results: walls joined (normal vs. switched), skipped, already joined, failed (with IDs).

Key Files

Commands/Handlers/JoinAllConcreteWallsAndFloorsGeometryHandler.cs
Design Strength
Excellent per-element error handling. Every operation is wrapped in try-catch. Failures are logged and reported but never stop the overall process. The switched-order retry handles Revit's unpredictable join-order sensitivity.
Warning Suppression
Uses SuppressWarningsPreprocessor to silently handle Revit's geometry join warnings that would otherwise halt the transaction.
Split Floor Ribbon: Structure → Split Floor
Solid

What It Does

Splits compound (multi-layer) floors into individual single-layer floors — the floor equivalent of Split Wall. Each layer becomes a separate floor at its correct vertical offset. Preserves slope arrows, shape editing (vertex modifications), and openings from the original floor.

Workflow

  1. Select one or more compound floors.
  2. Click Split Floor.
  3. Dialog shows layer breakdown with type-assignment combo per layer (auto-create or existing type).
  4. Choose scope: selected floors only, or all floors of the same type.
  5. Click Split — engine runs a 4-transaction pipeline per floor.

Property How It's Preserved

Key Files

Commands/Handlers/SplitFloorHandler.cs Helpers/FloorSplitter/FloorSplitterEngine.cs Helpers/FloorSplitter/FloorLayerAnalyzer.cs Views/FloorSplitterDialog.xaml
Opening Recreation — Z-Flattening

Sloped floors have opening curves at varying Z elevations. The engine flattens all opening curves to a consistent Z before recreation. Without this, openings on sloped floors would be created at the wrong elevation or fail entirely. Handles Line and Arc curves explicitly; other curve types pass through as-is.

Edge Case
Shape editing vertex matching uses XY proximity (3mm). Floors with very close vertices or irregular subdivision could mismatch. In practice, this tolerance works for standard construction geometry.
Boundary Fallback
If sketch profile extraction fails, the engine falls back to extracting edges from the topmost horizontal PlanarFace in the floor's geometry. Less precise but ensures the split doesn't abort entirely.
Join Beams to Walls Utility (no ribbon button)
Solid

What It Does

Joins geometry between selected structural elements (beams, columns, foundations) and all intersecting walls. Same join-retry pattern as Join Walls & Floors, but operates on structural framing instead of floors/roofs.

Workflow

  1. Select beams, columns, or structural foundations.
  2. Invoke the command (available as utility, not on the main ribbon).
  3. For each structural element:
    • Finds intersecting walls via ElementIntersectsElementFilter.
    • Skips already-joined, thin, curtain, and stacked walls.
    • Joins with retry on ArgumentException (switched order).
  4. Results dialog shows per-category breakdown (beams/columns/foundations processed, walls joined, failures).

Key Files

Commands/Handlers/JoinSelectedBeamsToBlockWallsHandler.cs
Shared Pattern
Nearly identical to Join Walls & Floors. Both use the same intersection filter, wall filtering (width/curtain/stacked), join-retry logic, and results reporting. Could be refactored into a shared base class, but both work correctly as-is.

Cross-Cutting Observations
  • Join commands (Walls&Floors + Beams) share 90% of their code. Both are well-engineered with per-element error isolation and the join-order retry pattern. A shared helper would eliminate duplication.
  • Split Floor is impressively complete — slope arrows, shape editing vertex transfer, and opening recreation with Z-flattening. More complex than Split Wall because floors carry more geometric metadata.
  • Beams To Ground is the weakest command — no rollback safety, deletes originals before confirming replacements. Could lose data on partial failure.
