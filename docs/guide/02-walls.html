Walls
10 commands — phase sync, copy, height, split, overlaps, cladding, trim corners, window families

The Walls panel is the largest group. It covers the full lifecycle of wall manipulation: copying from links, adjusting geometry, splitting compound walls into layers, detecting duplicates, and adding cladding. The Split Wall → Trim Corners pipeline is the most complex multi-step workflow in the entire add-in.

Sync Phases Ribbon: Walls → Sync Phases
Solid

What It Does

Sets the PHASE_CREATED parameter of selected windows and doors to match their host wall's phase. Useful after walls are reassigned to different phases — the hosted elements don't follow automatically.

Workflow

  1. Select one or more windows/doors (or select walls that host them).
  2. Click Sync Phases.
  3. The handler filters selection to windows and doors only.
  4. For each element, reads the host wall's PHASE_CREATED and copies it to the element.
  5. Elements with read-only phase parameters are skipped and reported.
  6. Shows summary: "Updated X elements, Y failed."

Key Files

Commands/Handlers/SetSelectedWindowAndDoorPhaseToWallPhaseHandler.cs
Note
Does not validate that the selection is non-empty — will silently show "0 updated" if nothing is selected.
Copy Linked Walls Ribbon: Walls → Copy Linked Walls
Solid

What It Does

Copies all walls from a selected Revit link instance into the host document, applying the link's transform so they land in the correct position.

Workflow

  1. Click Copy Linked Walls.
  2. Pick a Revit link instance in the viewport (custom LinkSelectionFilter).
  3. Handler retrieves all walls from the linked document.
  4. Uses ElementTransformUtils.CopyElements() with the link's total transform.
  5. Newly copied walls are selected in the UI.
  6. Shows count summary.

Wall To Floors Ribbon: Walls → Wall To Floors
Sketchy

What It Does

Auto-adjusts selected walls' base offset and top to sit flush against the nearest floors above and below. Uses ReferenceIntersector to cast rays from wall endpoints to find floor surfaces.

Workflow

  1. Select one or more walls.
  2. Click Wall To Floors.
  3. Handler finds (or falls back to) the default {3D} view for ray casting.
  4. For each wall:
    • Casts rays downward from wall base → finds floor below → adjusts base offset.
    • Casts rays upward from wall top → finds floor above → adjusts top offset or unconnected height.
  5. Shows completion summary.

Key Files

Commands/Handlers/AdjustWallHeightToFloorsHandler.cs
No Per-Wall Error Handling

The main loop has no try-catch per wall. If one wall has invalid geometry or a null LocationCurve, the entire operation crashes. Each iteration should be wrapped individually.

Assumes Linear Walls Only

Ray casting uses start/end points of the wall curve. Curved walls will only have rays at their endpoints, missing the mid-arc. Not necessarily wrong, but worth knowing.

Split Wall Ribbon: Walls → Split Wall
Solid

What It Does

Splits a compound (multi-layer) wall into individual single-layer walls. Each layer becomes a separate wall, offset to its correct position. Hosted elements (doors, windows) are transferred to the designated target layer. After splitting, saves results to JSON for the Trim Corners follow-up command.

Workflow

  1. Select one or more compound walls (non-compound walls are filtered out).
  2. Click Split Wall.
  3. Dialog shows layer breakdown: function, material, thickness, and a type-assignment combo per layer.
  4. For each layer, choose an existing WallType or select Auto-Create (creates a new type matching the layer's thickness).
  5. Choose which layer should receive hosted elements (Auto = thickest structural layer, or manual pick).
  6. Choose scope: selected walls only, or all walls of the same type in the project.
  7. Click OK — engine runs a 4-transaction pipeline per wall.
  8. Results saved to split_results.json. Summary dialog instructs you to run Trim Corners next.

Commands/Handlers/SplitWallHandler.cs Helpers/WallSplitter/WallSplitterEngine.cs Helpers/WallSplitter/CompoundLayerAnalyzer.cs Helpers/WallSplitter/WallJoinReplicator.cs Views/WallSplitterDialog.xaml
CompoundLayerAnalyzer — How Offsets Work

Each layer's center offset is computed from the wall's centerline (ignoring WALL_KEY_REF_PARAM). Formula: centerOffset = (totalWidth/2) - (runningDistance + thickness/2). Positive offset = toward exterior = wall.Orientation direction. Zero-thickness membrane layers are skipped.

Revit Gotcha
TransactionGroup.Assimilate() invalidates in-memory Element references. The engine stores ElementId values and re-fetches after assimilation.
File Dependency
Results are saved to split_results.json in the plugin DLL directory. If the directory is not writable (e.g., locked by another process), the save fails silently and Trim Corners won't work.
Trim Corners Dockable Panel → Trim Corners button
Solid

What It Does

The second step of the Split Wall pipeline. After walls are split into layers, the replacement walls don't connect at corners. This command reads the JSON output from Split Wall and cross-joins replacement walls at their intersection points.

Workflow

  1. Run Split Wall first (produces split_results.json).
  2. Click Trim Corners in the dockable panel.
  3. Handler reads JSON, re-fetches walls from document (validates they still exist).
  4. Two-pass corner trimming:
    • Pass 1: Join replacement walls between split groups by matching layer index (outer↔outer, inner↔inner).
    • Pass 2: Trim each replacement against nearby unsplit neighbor walls.
  5. JSON file is deleted after success.

Key Files

Commands/Handlers/TrimWallCornersHandler.cs Helpers/WallSplitter/WallJoinReplicator.cs Views/TrimCornersView.xaml
Corner Detection Algorithm

Uses 2D line-line intersection on wall centerlines. An intersection is considered a "corner" if the intersection point is within 500mm of an endpoint on both walls. Parallel walls or distant intersections are ignored. End-join states are toggled (allow→disallow→allow) to force geometry refresh.

Dependency
Completely depends on split_results.json from the Split Wall step. If the file is missing, shows "Run Split Wall first."
Thread Safety
Runs via CommandDispatcherService (queued from the dockable panel button), not directly from a ribbon command.
Detect Overlaps Ribbon: Walls → Detect Overlaps
Solid

What It Does

Finds walls of the same type that overlap each other. Groups them, calculates overlap percentage, and lets you highlight, unjoin, or delete the duplicates. 100% overlaps (exact duplicates) sort to the top.

Workflow

  1. Optionally select walls to scan (if none selected, scans all walls in active view).
  2. Click Detect Overlaps.
  3. Detector groups walls by type, then checks pairwise:
    • Are they parallel? (angular tolerance ~1°)
    • Is perpendicular distance < 50% of wall width?
    • Same base level?
    • Overlap length ≥ 50mm?
  4. Overlap % computed via 3D solid boolean intersection (falls back to 2D estimate if geometry fails).
  5. Results dialog shows groups with: overlap%, type, wall count, IDs, and suggested "Keep" wall.
  6. Choose action:
    • Highlight: Selects and temporarily isolates overlapping walls.
    • Unjoin: Unjoins all pairs in each group.
    • Delete Duplicates: Keeps the wall with the most hosted elements, deletes the rest.

Key Files

Commands/Handlers/DetectOverlappingWallsHandler.cs Helpers/OverlappingWallDetector/OverlappingWallsDetector.cs Views/OverlappingWallsResultDialog.xaml
Design Strength
Boolean geometry operation (BooleanOperationsUtils) wrapped in try-catch with automatic fallback to 2D width-based estimate. Handles degenerate geometry gracefully.
Performance
Pairwise comparison within each type group is O(n²). Fine for typical projects, but could be slow with hundreds of walls of the same type.
Wall Cladding (3 variants) Ribbon: Walls → Cladding ▾ (Both / External / Internal)
Sketchy

What It Does

Adds cladding (chipuy) walls parallel to selected walls. Three variants in a split button: Both Sides, External Only, and Internal Only. Each creates offset walls using predefined cladding wall types, then joins geometry and connects end-join corners.

Workflow

  1. Click one of the three cladding buttons.
  2. Interactive wall picker — select walls to clad.
  3. For each wall:
    • Calculates offset from wall orientation and thickness.
    • Creates parallel cladding walls at computed positions.
    • Sets location line: FinishFaceInterior (external) or FinishFaceExterior (internal).
    • Copies parameters from original wall.
  4. Second transaction: joins cladding walls to originals.
  5. Third transaction: connects end-join corners between cladding segments.

Key Files

Commands/Handlers/AddChipuyToWallHandler.cs Commands/Handlers/AddChipuyToExternalWallHandler.cs Commands/Handlers/AddChipuyToInternalWallHandler.cs Helpers/Cladding/CladdingWallCreator.cs
Hardcoded Wall Type Names

CladdingWallCreator resolves cladding types by name ("w1" for external, "w2" for internal). If these types don't exist in the project, it throws InvalidOperationException with no recovery path. Should either prompt for type selection or create the types automatically.

Curved Walls Not Supported

Offset calculation assumes linear wall curves. Curved walls will get incorrect cladding placement (the offset vector is computed from wall.Orientation, which is a single direction).

Cross-Cutting Observations
  • Split Wall + Trim Corners is the most robust pipeline in the codebase — 4-transaction design with rollback, hosted element transfer, join replication, and a two-pass corner algorithm. Well-engineered.
  • Cladding is the weakest — hardcoded type names, no curved wall support, three near-identical handler classes that could be a single handler with a mode parameter.
  • All wall handlers assume linear walls. Curved wall support is not present anywhere in this panel.
