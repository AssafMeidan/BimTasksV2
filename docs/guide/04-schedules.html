Schedules
3 commands — export, roundtrip editing, key schedule import

The Schedules panel bridges Revit schedules and Excel. Two services power these commands: ScheduleExcelExportService for professional one-way export with full formatting, and ScheduleExcelRoundtripService for bidirectional editing with element tracking. All three commands share the SchedulePickerWindow for schedule selection.

Schedule Filtering
All three commands filter schedules before showing the picker. Excluded: titleblock revision schedules, internal keynote schedules, key schedules, and system schedules (names starting with <).
Export Schedule Ribbon: Schedules → Export Schedule
Solid

What It Does

Exports one or more schedules to professionally formatted Excel files. Supports two modes: all selected schedules in a single workbook (one sheet per schedule), or each schedule as a separate .xlsx file in a folder.

Workflow

  1. Click Export Schedule.
  2. SchedulePickerWindow opens in multi-select mode with search filter.
  3. Select schedules (Ctrl+Click for multiple). Choose export mode: "One Excel file" or "Separate files."
  4. Single file: Save dialog → all schedules written as sheets in one workbook.
  5. Separate files: Folder picker → one .xlsx per schedule, filenames sanitized.
  6. Shows success summary with file count.

Edit In Excel Ribbon: Schedules → Edit In Excel
Solid

What It Does

Full roundtrip: exports a schedule to a temp Excel file, opens it for editing, waits for the user to save and close, then imports changes back into Revit. Writable cells have white backgrounds; read-only cells are grayed out.

Edit In Excel — Roundtrip Flow

Workflow

  1. If active view is a schedule, it's used directly. Otherwise, schedule picker opens (single-select).
  2. Schedule exported to temp file: %TEMP%\BimTasksV2_{name}_{timestamp}.xlsx
  3. Excel opens automatically via shell association.
  4. Revit shows a confirmation dialog: "Edit, save, close Excel, then click Import Changes."
  5. On Import: reads Excel back, compares each cell to current Revit value.
  6. Changed cells update the element's parameter (instance or type parameter).
  7. Results summary: updated cells, unchanged (skipped), failed, affected elements.
  8. Temp file deleted (best-effort).

How Element Tracking Works

The roundtrip service writes a hidden column A containing comma-separated ElementId values for each row. On import, it reads column A to find the exact elements to update. For itemized schedules, rows map 1:1 to elements. For grouped schedules, an element signature (pipe-delimited field values) groups elements by their shared properties.

Key Files

Commands/Handlers/EditScheduleInExcelHandler.cs Services/ScheduleExcelRoundtripService.cs Views/SchedulePickerWindow.xaml
No Verification That Excel Was Saved

The handler trusts the user to save and close Excel before clicking "Import Changes." If the user forgot to save, the import reads the last-saved state (or the original export). There's no timeout, no file-modification check, and no warning if the file hasn't changed.

Process.Start Without Error Handling

Process.Start() with UseShellExecute = true will throw if Excel (or any .xlsx handler) is not installed. This exception is not specifically caught — it falls through to the generic error handler.

Import Key Schedules Ribbon: Schedules → Import Key Schedules
Sketchy

What It Does

Imports key schedule data from an Excel file into 4 Revit key schedules (Walls, Floors, Structural Columns, Structural Framing). Creates the key schedules if they don't exist. Each row in Excel becomes a key entry with 5 fields: KeyName, ItemNumber, Description, Price, Subcontractor.

Workflow

  1. Click Import Key Schedules.
  2. Open file dialog → select .xlsx file.
  3. Reads first worksheet, starting at row 2 (row 1 = header). 5 columns: KeyName, ItemNumber, Description, Price, Subcontractor.
  4. For each of 4 target categories:
    • Finds or creates BIMTasks_Key_{Category} schedule.
    • Ensures all parameter fields exist on the schedule.
    • For each Excel row: finds existing key element by name or inserts new row.
    • Sets all 5 parameters on the element.
  5. Shows success dialog.

Key Files

Commands/Handlers/ImportKeySchedulesFromExcelHandler.cs
No Excel Format Validation

Assumes the Excel file has exactly 5 columns in a specific order. No header validation, no column count check. If the user opens a differently structured file, data will be silently misassigned (e.g., Description written to Price column).

Silent Price Parse Failure

Price column is parsed with double.TryParse() — non-numeric values silently default to 0. No warning shown to the user that pricing data was lost.

No Duplicate Key Detection

If two Excel rows have the same KeyName, both will be processed. The second row overwrites the first without warning. Should either merge, skip, or warn.

Shared UI — Schedule Picker

All three commands use SchedulePickerWindow, a reusable WPF dialog with two modes:

Single-Select Mode
Used by Edit In Excel and Import Key Schedules. One schedule at a time. Double-click to confirm. No export mode radio buttons.
Multi-Select Mode
Used by Export Schedule. Ctrl+Click for multiple. Shows radio buttons: "One Excel file" (multiple sheets) vs. "Separate Excel files" (one per schedule).

Both modes include a real-time search box that filters the schedule list by name (case-insensitive). Schedules are sorted alphabetically.

Cross-Cutting Observations
  • Export quality is professional-grade — 27 cell styles, auto-detected column types, summary row classification, RTL support, and print-ready layout. This is well above typical Revit add-in export quality.
  • Roundtrip editing is clever — hidden ElementId column + writable/read-only field analysis + element signature matching. The complexity is justified by the use case.
  • Key schedule import is the weakest link — no format validation, silent data loss on parse failures, and no duplicate detection. Needs hardening before relying on it for production data.
  • Two separate export services exist: ScheduleExcelExportService (presentation-grade, one-way) and ScheduleExcelRoundtripService (element-tracked, bidirectional). They share no code but serve distinct purposes.
